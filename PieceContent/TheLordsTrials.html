The Lord's Trials

An escape room style game made in Unity.

📋 Role Programmer
⌚ Duration 3 Weeks
👩‍💻 Team Size 2
⚙ Tools Unity
Visual Studio
Git
Adobe Photoshop

Overview

This project was a submission for the Indie City Games Chicaghoul 2025 game jam. The following was created with the theme of a haunted castle and The Yellow King prompt. For this project I designed, collaborated, programmed, and delegated tasks to ensure the game met the required specifications and was completed within the allotted time.

Goal

Since this was my second participation in a game jam, I aimed to improve upon my previous experience and deliver a more polished and engaging product. For this, we aimed for an ambitious scope while maintaining a focus on gameplay and player experience.

Task Responsibilities

  • Designed and implemented UI menus using Unity's UI Toolkit. A total of 5 menu screens were created. Along with creating a UI Management script to allow for dynamic UI updates.
  • Implemented selected puzzles with their respected mechanics and win conditions.
    • Potion Order Selection
    • Basket Item Scale
  • Created visuals to handle different effects, feedback for the player and overall aesthetic.
    • Scene transitions
    • Baked Lighting
    • Particle Effects: Godray particles & Force Field Shield
    • 2D backgrounds and UI: made using Adobe Photoshop
  • Delegating tasks to team members to ensure efficient workflow, tracking progress, and maintaining clear communication throughout the project. This included assigning specific tasks, setting deadlines, and providing support and guidance as needed.
  • Researching and collecting assets for the game to fit the overall theme and style.

Challenges

One of the main challenges I encountered was working with the potion puzzle. The idea was to have the player select the correct options to be poured into the cauldron. However, I wasn't able to have different colors change based on the selected options. Rather than implementing this feature, I focused on ensuring the core mechanics were solid. Thus, a workaround was to simply play the pour animation, with the cauldron being opaque. It was also placed higher than the player so they wouldn't be able to look inside.

Another small challenge I encountered was overcomplicating the basket item scale puzzle. It originally consisted of checking the weight of the item to be placed on a basket and if it's the correct type. However, it didn't provide much use to have each items weight differently. Additionally, the basket was going to be idle. To make it easier to implement, I simplified the puzzle to only check if the item is the correct type. The basket would also be animated to move and down when an item is placed on it. And if there was an incorrect item, it'll dump the objects and return back to the original position. This provided a more dynamic and interactive experience for the player.

Overall, the main challenge was to ensure that the game was completed within the time frame. This required careful planning, prioritization of tasks, and effective communication among team members. By staying focused and organized, we were able to overcome these challenges and deliver the final product.

Results

Overall, the game was completed within the time frame and met the required specifications. We did have some minor issues from players not passing certain puzzles or taking longer to solve. I enjoyed developing the game and playtesting it. My favorite part was creating a unique combination result for the potions puzzle. That way, each run was different rather than becoming predictable and repetitive. In the end, the game provided a fun and engaging experience for players and replayability to get the best ending.
